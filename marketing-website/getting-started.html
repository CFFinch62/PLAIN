Getting Started with PLAIN

Install PLAIN and write your first program in minutes

Prerequisites

Before you begin, make sure you have:

  • Go 1.21 or later - PLAIN is implemented in Go
  • Git - To clone the repository
  • A text editor - Any editor will work (VS Code, Sublime, Vim, etc.)
  • Terminal/Command Line - Basic familiarity recommended

Installation

1

Clone the Repository

First, clone the PLAIN repository from GitHub:

git clone https://github.com/yourusername/PLAIN.git
cd PLAIN

Replace yourusername with the actual repository location.

2

Build the Interpreter

Build the PLAIN interpreter using Go:

go build -o plain cmd/plain/main.go

This creates a plain executable in the current directory.

3

Verify Installation

Test that PLAIN is working:

./plain --version

✓ If you see version information, you're ready to go!

Your First PLAIN Program

1

Create a New File

Create a file named hello.plain with the following content:

hello.plain 
rem: My first PLAIN program

task Main()
    var strName = "World"
    var greeting = v"Hello, {strName}!"
    display(greeting)
    display("Welcome to PLAIN programming!")
2

Run Your Program

Execute your program with the PLAIN interpreter:

./plain run hello.plain

You should see:

Hello, World!
Welcome to PLAIN programming!

✓ Congratulations! You've written and run your first PLAIN program!

Quick Tips

💡 Use Type Prefixes

Start variable names with type prefixes (intAge, strName) to make your code self-documenting.

📝 Comment Your Code

Use rem: for single-line comments and note: for multi-line comment blocks.

🎯 One Task, One Purpose

Keep tasks focused on a single responsibility. Break complex logic into smaller, named tasks.

🔍 Read Error Messages

PLAIN's error messages are designed to be helpful. They explain what went wrong and how to fix it.

Common Issues

Build fails with "Go not found"

Solution: Install Go from golang.org/dl and make sure it's in your PATH.

Program won't run - "Main task not found"

Solution: Every PLAIN program needs a task Main() as the entry point. Make sure you've defined it.

Indentation errors

Solution: PLAIN uses indentation to define blocks (like Python). Use consistent spaces (4 spaces recommended) for indentation.

Variable shadowing error

Solution: PLAIN doesn't allow variable shadowing. If you see this error, you're trying to redeclare a variable that already exists in an outer scope. Use a different name or assign to the existing variable without var.
