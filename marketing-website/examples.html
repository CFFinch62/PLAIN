PLAIN Code Examples

See how clear, capable code looks in practice

These examples demonstrate PLAIN's natural syntax, clear structure, and powerful features. Each example is complete and runnable - copy, paste, and try them yourself!

Hello World

Beginner

The classic first program. Notice how natural the syntax is - it reads like instructions.

hello.plain 
rem: Simple Hello World program in PLAIN

task Main()
    var strName = "World"
    var greeting = v"Hello, {strName}!"
    display(greeting)

Key Takeaways

  • task Main() is the entry point
  • var declares new variables
  • v"..." enables string interpolation
  • display() outputs to console

Fibonacci Sequence

Beginner

Recursive function demonstrating tasks, parameters, and return values.

fibonacci.plain 
rem: Calculate Fibonacci numbers

task Fibonacci using (intN)
    if intN <= 1
        deliver intN
    
    var a = Fibonacci(intN - 1)
    var b = Fibonacci(intN - 2)
    deliver a + b

task Main()
    var intCount = 10
    
    loop i from 0 to intCount
        var result = Fibonacci(i)
        display(v"Fibonacci({i}) = {result}")

Key Takeaways

  • using declares parameters
  • deliver returns values
  • loop from/to creates numeric ranges
  • Recursion works naturally

Working with Lists

Beginner

Demonstrates list operations, iteration, and the standard library.

lists.plain 
task Main()
    rem: Create and populate a list
    var lstNumbers = [1, 2, 3, 4, 5]
    
    rem: Add items
    append(lstNumbers, 6)
    append(lstNumbers, 7)
    
    rem: Iterate and process
    var total = 0
    loop num in lstNumbers
        total = total + num
    
    var average = total / length(lstNumbers)
    
    display(v"Total: {total}")
    display(v"Average: {average}")
    display(v"Count: {length(lstNumbers)}")

Key Takeaways

  • Lists use square bracket syntax []
  • loop in iterates over collections
  • Standard library functions like append() and length()
  • Variables can be reassigned without var

Error Handling

Intermediate

Shows PLAIN's built-in error handling with attempt/handle/ensure blocks.

error_handling.plain 
task SafeDivide using (a, b)
    attempt
        var result = a / b
        deliver result
    handle
        display("Error: Division failed")
        deliver null
    ensure
        display("Division operation completed")

task Main()
    rem: Test with valid division
    var result1 = SafeDivide(10, 2)
    display(v"10 / 2 = {result1}")

    rem: Test with division by zero
    var result2 = SafeDivide(10, 0)

    if result2 == null
        display("Division was not possible")

Key Takeaways

  • attempt block contains code that might fail
  • handle block runs if an error occurs
  • ensure block always runs (like finally)
  • Error handling is built into the language

Records (Custom Types)

Intermediate

Define custom data structures with records and use them in type-safe collections.

records.plain 
rem: Define a Person record
record Person:
    name as string
    age as integer = 0
    email as string = ""

task Main()
    rem: Create person instances
    var alice = Person(
        name: "Alice",
        age: 25,
        email: "alice@example.com"
    )

    var bob = Person(
        name: "Bob",
        age: 30,
        email: "bob@example.com"
    )

    rem: Type-safe list of people
    var people as list of Person = [alice, bob]

    rem: Iterate and display
    loop person in people
        display(v"{person.name} is {person.age} years old")

Key Takeaways

  • record defines custom data types
  • Fields can have default values
  • Named parameters make construction clear
  • Type-safe collections with list of Type

Loop with Step

Beginner

Demonstrates the step keyword for custom loop increments.

loop_step.plain 
task Main()
    rem: Count by 5s
    display("Count by 5s from 0 to 50:")
    loop i from 0 to 50 step 5
        display(v"  {i}")

    rem: Countdown with negative step
    display("Countdown by 2s from 10 to 0:")
    loop i from 10 to 0 step -2
        display(v"  {i}")

    rem: Even numbers only
    display("Even numbers from 2 to 20:")
    loop i from 2 to 20 step 2
        display(v"  {i}")

Key Takeaways

  • step keyword controls loop increment
  • Negative steps create countdown loops
  • Ranges are inclusive (include both start and end)
  • Natural syntax: loop i from 0 to 10 step 2
