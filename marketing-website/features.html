Features That Make PLAIN Unique

Designed for clarity without sacrificing capability

📖 Natural, Readable Syntax

Code that reads like plain English instructions

PLAIN uses natural language keywords that make code immediately understandable. Instead of cryptic symbols and abbreviations, you write code that reads like instructions you'd give to another person.

  • task - Define functions with clear purpose
  • deliver - Return values explicitly
  • attempt/handle/ensure - Error handling that makes sense
  • loop from/to/in - Iteration that reads naturally

No more wondering what def, fn, or func means - it's just task.

natural_syntax.plain 
task CalculateAverage using (numbers)
    var total = 0
    
    loop num in numbers
        total = total + num
    
    var average = total / length(numbers)
    deliver average

task Main()
    var scores = [95, 87, 92, 88]
    var avg = CalculateAverage(scores)
    display(v"Average: {avg}")
explicit_vars.plain 
task Example()
    var counter = 0    rem: declares new variable
    
    loop i from 1 to 10
        counter = counter + 1  rem: assigns to existing
    
    display(counter)  rem: 10

rem: This would be an ERROR:
task BadExample()
    var x = 5
    
    if x > 0
        var x = 10  rem: ERROR! No shadowing

🎯 Explicit Over Implicit

No surprises, no hidden behavior

PLAIN makes everything explicit. var declares a new variable, no var assigns to an existing one. This simple rule eliminates entire classes of bugs.

  • No shadowing - Can't redeclare variables in inner scopes
  • Clear declarations - Always know when you're creating vs. modifying
  • Explicit types - Optional but available when you need them
  • No implicit conversions - Types don't change behind your back

Code that's explicit is code that's maintainable.

🏷️ Self-Documenting Type Prefixes

Optional prefixes that make code crystal clear

PLAIN supports Hungarian-style type prefixes that make variable types immediately obvious. They're optional, but when you use them, your code becomes self-documenting.

  • int - Integer values (intAge, intCount)
  • flt - Floating point numbers (fltPrice, fltTemperature)
  • str - Strings (strName, strMessage)
  • bln - Booleans (blnActive, blnIsReady)
  • lst - Lists (lstItems, lstScores)
  • tbl - Tables/Dictionaries (tblData, tblConfig)

One glance tells you everything you need to know about a variable.

type_prefixes.plain 
task ProcessOrder()
    var intOrderId = 12345
    var strCustomer = "Alice"
    var fltTotal = 99.99
    var blnPaid = false
    var lstItems = ["Book", "Pen"]
    var tblPrices = {
        "Book": 89.99,
        "Pen": 9.99
    }
    
    rem: Types are obvious at a glance!
error_handling.plain 
task SafeDivide using (a, b)
    attempt
        var result = a / b
        deliver result
    handle
        display("Error: Division failed")
        deliver null
    ensure
        display("Operation completed")

task Main()
    var result = SafeDivide(10, 0)

    if result == null
        display("Division was not possible")

🛡️ Built-in Error Handling

First-class error handling that makes sense

PLAIN includes error handling as a core language feature with attempt, handle, and ensure blocks. No exceptions to catch, no try-catch confusion.

  • attempt - Try code that might fail
  • handle - Deal with errors gracefully
  • ensure - Code that always runs (like finally)
  • Clear flow - Error paths are explicit and obvious

Error handling that's built into the language, not bolted on.

📝 String Interpolation

Natural string formatting without the hassle

Format strings naturally with the v"..." syntax. No printf formats to memorize, no string concatenation chains to debug.

  • Simple syntax - Just v"Hello {name}!"
  • Any expression - Put calculations right in strings
  • Readable - See exactly what the output will look like
  • No escaping - Regular strings use normal quotes

String formatting that just works.

interpolation.plain 
task Main()
    var strName = "Alice"
    var intAge = 25
    var fltScore = 95.5

    rem: Simple interpolation
    display(v"Hello, {strName}!")

    rem: Multiple values
    display(v"{strName} is {intAge} years old")

    rem: Expressions work too
    display(v"Score: {fltScore * 2}")

    rem: No concatenation needed!
records.plain 
record Person:
    name as string
    age as integer = 0
    email as string = ""

task Main()
    var alice = Person(
        name: "Alice",
        age: 25,
        email: "alice@example.com"
    )

    display(alice.name)
    display(alice.age)

    rem: Type-safe collections
    var people as list of Person = []
    people.append(alice)

🔧 Records & Custom Types

Define your own data structures

Create custom data types with records. Define fields with types and default values. Build type-safe collections that hold exactly what you expect.

  • Clear syntax - Define fields with types and defaults
  • Type safety - Collections can be constrained to specific types
  • Named parameters - Create instances with clear field names
  • Dot notation - Access fields naturally

Build complex data structures without complexity.

📦 Clear Module System

Organize code with packages, assemblies, and modules

PLAIN's module system is hierarchical and explicit. Import exactly what you need with clear declarations.

  • Packages - Top-level organization
  • Assemblies - Collections of related modules
  • Modules - Individual code files
  • Selective imports - Import specific tasks or whole modules

Code organization that scales from small scripts to large projects.

imports.plain 
use:
    assemblies:
        System
        System.Collections
    modules:
        io
        io.files
        math
        string.utils
    tasks:
        io.files.ReadBinary
        math.advanced.CalculatePI

task Main()
    var data = ReadBinary("data.bin")
    var pi = CalculatePI(100)
    display(pi)
